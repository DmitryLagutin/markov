# 🔤 Markov Chain Text Generator

Python-генератор текста на основе цепей Маркова. Создает уникальные фразы на основе исходного текста.

## 🎯 Описание проекта

Этот инструмент использует алгоритм **Марковских цепей** для генерации нового текста на основе образца. Полезно для:
- **Генерации контента** - создание статей, постов, заголовков
- **Поиска вдохновения** - новые идеи и формулировки
- **Анализа стиля** - изучение языковых паттернов
- **Творческих экспериментов** - необычные текстовые комбинации

### Ключевые возможности:

✅ Построение матрицы переходов из слов  
✅ Случайная генерация новых последовательностей  
✅ Поддержка русского и английского языка  
✅ Настройка длины генерируемого текста  
✅ Фильтрация по регистру (пробуждение от заглавных)

## 📋 Использование

### Базовый запуск

```bash

```

## 🔍 Как работает алгоритм

### 1. Создание пар слов

```python
def make_pairs(ind_words):
    for i in range(len(ind_words) - 1):
        yield (ind_words[i], ind_words[i + 1])
```

Пример для текста `"А Б В"`:
```
Пары: ('А', 'Б'), ('Б', 'В')
```

### 2. Построение словаря переходов

```python
word_dict = {}
for word_1, word_2 in pair:
    if word_1 in word_dict.keys():
        word_dict[word_1].append(word_2)
    else:
        word_dict[word_1] = [word_2]
```

Результат:
```python
{
    'А': ['Б'],
    'Б': ['В'],
    ...
}
```

### 3. Генерация фразы

```python
chain = [first_word]
while len(chain) < n_words:
    next_word = random.choice(word_dict[chain[-1]])
    chain.append(next_word)
```

## ⚠️ Важные замечания

### Качество результата

Качество зависит от:
1. **Объема обучающего текста** - чем больше, тем лучше
2. **Разнообразия контекста** - разные стили создают разные результаты
3. **Длины исходных предложений** - короткие фразы дают более хаотичные результаты

### Грамматика и смысл

Алгоритм Марковой Цепи:
- ❌ Не понимает грамматику
- ❌ Не знает смысла слов
- ✅ Анализирует частоту совпадений
- ✅ Создает правдоподобные последовательности

### Языковая специфика

Для лучшего результата:
- Используйте единый регистр (все нижний или все верхний)
- Убедитесь что пробелы правильно разделены
- Избегайте специальных символов (можно удалить перед анализом)

## 🐛 Решенные проблемы

### Проблема: Пустой вывод

**Причина:** Нет подходящих переходов в словаре

**Решение:**
1. Увеличьте объем входного текста
2. Проверьте что файл существует и заполнен
3. Добавьте fallback к начальным словам

### Проблема: Бесконечный цикл

**Причина:** Цикл в графе переходов

**Решение:**
```python
max_iterations = 1000
count = 0
while count < max_iterations and len(chain) < n_words:
    # ...
    count += 1
```

### Проблема: Плохие результаты с русским текстом

**Решение:** Проверьте кодировку:

```python
data = open('new.txt', encoding='utf-8').read()
```

## 💡 Расширение функциональности

### Н-граммы (N-grams)

Использование больгьших контекстных окон:

```python
def make_ngram_pairs(words, n=2):
    for i in range(len(words) - n + 1):
        key = tuple(words[i:i+n])
        value = words[i+n] if i+n < len(words) else None
        yield key, value
```
